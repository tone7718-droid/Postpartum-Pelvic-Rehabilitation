# 일러스트 · 인터랙션 설계 명세 (Web 구축용)

> 목적: 전자책을 Next.js 웹사이트로 만들 때 필요한 **시각 요소(일러스트)**와 **인터랙션(컴포넌트)**을 미리 정의합니다.
> 원칙: ① 따뜻하고 안심되는 톤 ② 텍스트 없이도 동작을 이해할 수 있는 그림 ③ 안전 메시지를 항상 동반 ④ 접근성(큰 글씨·고대비·스크린리더).

---

## 1. 비주얼/브랜드 방향

- **톤**: 부드럽고 차분한 산후 케어 무드. 의학적이지만 위협적이지 않게.
- **컬러(초안)**: 메인 `#E8A6B8`(연한 로즈), 서브 `#7BA6A1`(세이지), 배경 `#FBF7F4`(웜 화이트), 텍스트 `#3A3A3A`. 경고는 `#C0524B`(차분한 레드), 정보는 `#5B7C99`.
- **타이포**: 한글 가독성 우선(예: Pretendard). 본문 17~18px, 줄간격 1.7 이상.
- **일러스트 스타일**: 선이 부드러운 플랫 일러스트, 동작은 **측면/3/4 뷰**로 관절 움직임이 보이게. 다양한 체형 포용.
- **아이콘 규칙**: ⚠️ 주의 · ⛔ 금지 · 👀 자가관찰 · 💡 팁 · 🧭 내비 — 콘텐츠의 이모지 체계를 컴포넌트로 표준화.

## 3. 인터랙티브 컴포넌트

| ID | 컴포넌트 | 동작 | 연결 콘텐츠 |
|---|---|---|---|
| IX-Assess | **셀프 평가 인터랙티브 체크** | 1장 검사 문항을 단계별로 응답 → 결과로 "일반 진행 가능 / 전문가 평가 권장" 안내(진단 아님 명시) | 1장 |

| IX-Routine | **하루 10분 루틴 플레이어** | 카드들을 순서대로 진행, 타이머·다음동작 자동 전환 | 2장 2-6 |
| IX-Timeline | **회복 타임라인 슬라이더** | 현재 주차 입력→해당 시기 "해도 되는 것/피할 것" 표시 | 3장 |
| IX-Return | **고충격 복귀 준비도 체크** | 12주+ 기준 체크리스트 통과 여부 판정 | 3장 3-5 / 5장 |

## 4. 데이터·프라이버시 메모

- **증상 일지(IX-Diary)는 민감 건강정보**입니다. 1차적으로 **브라우저 로컬(localStorage/IndexedDB)** 저장을 기본으로 하고, 서버 저장(Supabase 등)은 **명시적 동의 + 인증**이 있을 때만.
- 모든 인터랙티브 결과 화면에 **"교육용 선별 도구이며 진단이 아님 / 증상 시 전문가 상담"** 문구 고정 노출.

---

## 5. 기술 메모 (Next.js)

- **콘텐츠 소스**: `content/*.md`를 MDX로 렌더(장별 라우트). 표·콜아웃·이모지 콜아웃을 커스텀 컴포넌트로 매핑.
- **컴포넌트화**: 위 IX-* 를 React 컴포넌트로, MDX 안에서 `<ExerciseCard/>`, `<SelfCheck/>` 형태로 삽입.
- **접근성**: 시맨틱 HTML, 키보드 내비, prefers-reduced-motion 존중(애니메이션 최소 옵션).
- **반응형**: 모바일 우선(수유 중 한 손 사용 가정 — 큰 탭 타깃).
- **일러스트 포맷**: SVG(경량·확대 또렷). 초기에는 플레이스홀더 → 점진 교체.

---

## 6. 제작 순서 제안

1. **MDX 렌더 + 장 라우팅 + 내비게이션(IX-Nav)** — 텍스트 전자책이 먼저 읽히게
2. **운동 카드/루틴 플레이어(IX-Exercise/Routine)** + 핵심 일러스트(IL-02*) — 가장 자주 쓸 기능
3. **셀프 평가·타임라인(IX-Assess/Timeline)** + IL-01*/IL-03a
4. **증상 일지·체크리스트(IX-Diary/Checklist)**
5. 나머지 일러스트·용어 툴팁·다듬기 → Vercel 배포

> 이 순서는 "읽을 수 있는 책 → 따라 할 수 있는 운동 → 점검 도구"로 가치가 빨리 나오게 한 것입니다.
